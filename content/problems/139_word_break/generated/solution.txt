# Word Break Solution

## Approach: Dynamic Programming

The idea is to use a boolean DP array where `dp[i]` represents whether the substring `s[0:i]` can be segmented using words from the dictionary.

### Algorithm:
1. Create a DP array of size `n+1` initialized to `False`
2. Set `dp[0] = True` (empty string is always valid)
3. For each position `i`, check all possible substrings ending at `i`
4. If `dp[j]` is `True` and `s[j:i]` is in the dictionary, set `dp[i] = True`

### Visualization:
```

```

## Code

```python
from typing import List

class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        """
        Dynamic Programming approach to solve Word Break problem.
        
        Args:
            s: Input string to be segmented
            wordDict: List of valid dictionary words
            
        Returns:
            True if s can be segmented into dictionary words, False otherwise
            
        Time Complexity: O(n^2) where n is length of string s
        Space Complexity: O(n) for the DP array
        """
        word_set = set(wordDict)  # Convert to set for O(1) lookup
        n = len(s)
        
        # dp[i] = True means s[0:i] can be segmented using dictionary words
        dp = [False] * (n + 1)
        dp[0] = True  # Empty string is always valid
        
        for i in range(1, n + 1):
            for j in range(i):
                # Check if s[0:j] is valid AND s[j:i] is in dictionary
                if dp[j] and s[j:i] in word_set:
                    dp[i] = True
                    break  # No need to check further once dp[i] is True
        
        return dp[n]


# Test cases
```

## Complexity Analysis

| Complexity | Value | Reason |
|------------|-------|--------|
| Time | O(n²) | Two nested loops over string length n |
| Space | O(n) | DP array of size n+1 |

> **Key Insight**: Converting `wordDict` to a set reduces each word lookup from O(m) to O(1), where m is the dictionary size. The `break` statement optimizes by stopping early once a valid segmentation is found for position `i`.
