## Sliding Window Maximum Solution

### Approach: Monotonic Deque

The key insight is to use a **monotonic decreasing deque** that stores indices of elements. The deque maintains the property that elements are in decreasing order, so the front always contains the index of the maximum element in the current window.

**Algorithm:**
1. Use a deque to store indices of elements in decreasing order of their values
2. For each new element:
   - Remove indices from the back that have smaller values (they can never be maximum)
   - Remove indices from the front that are outside the current window
   - Add current index to the back
3. After processing the first `k` elements, the front of the deque is always the maximum

### Code

```python
from collections import deque
from typing import List

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        """
        Returns the maximum values for each sliding window of size k.
        
        Args:
            nums: List of integers
            k: Window size
            
        Returns:
            List of maximum values for each window position
        """
        if not nums or k == 0:
            return []
        
        result = []
        dq = deque()  # Stores indices, maintains decreasing order of values
        
        for i in range(len(nums)):
            # Step 1: Remove elements outside the current window from the front
            # If the front index is out of window bounds, remove it
            while dq and dq[0] < i - k + 1:
                dq.popleft()
            
            # Step 2: Remove smaller elements from the back
            # If back element is smaller than current, it can never be maximum
            while dq and nums[dq[-1]] < nums[i]:
                dq.pop()
            
            # Step 3: Add current index to the back
            dq.append(i)
            
            # Step 4: Start adding results once the first window is complete
            if i >= k - 1:
                result.append(nums[dq[0]])  # Front is always the maximum
        
        return result


# Test cases
```

### Complexity Analysis

| Complexity | Value | Reason |
|------------|-------|--------|
| **Time** | O(n) | Each element is added and removed from deque at most once |
| **Space** | O(k) | Deque stores at most k indices at any time |
